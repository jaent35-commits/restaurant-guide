# 홈 접근 비밀번호 게이트 — 설정 가이드

앱 진입 시(홈 포함 앱 전체) 비밀번호를 요구하는 게이트입니다. 검증은 **Cloudflare Worker(서버)에서만** 수행하며, 비밀번호 원문은 어떤 화면·API 응답에도 노출되지 않습니다. 값 변경은 **노션에서 직접** 합니다.

## 동작 요약

- **적용 범위**: 앱 전체. 잠금 상태에서는 홈/식당정보/충전/사용 어떤 화면도 열리지 않고, 데이터도 로드되지 않습니다.
- **세션 유지**: 한 번 통과하면 **2시간** 유지(localStorage 저장). 앱/브라우저를 닫았다 다시 열어도 2시간 이내면 재입력 불필요. 2시간이 지나면 자동으로 잠기고 다시 요구합니다.
- **fail-closed**: 서버(Worker)에 연결할 수 없거나 검증이 불가능하면 통과시키지 않습니다.
- **입력 UX**: 기본 마스킹, 우측 **뜬눈/감은눈 아이콘 버튼**으로 보이기/숨기기 토글.
- **초기 비밀번호**: `53075`

## 노션 테이블

최초 검증 시 Worker가 노션에 전용 테이블을 **자동 생성**하고 초기 비밀번호(`53075`)를 넣습니다.

- 테이블(DB) 제목: **`앱 접근 비밀번호`**
- 속성:
  - `이름` (제목) — 라벨용(예: "홈 화면 비밀번호")
  - `비밀번호` (텍스트) — **실제 비밀번호 값**
- 생성 위치: 기존 식당 DB의 부모 페이지 아래

### 비밀번호 변경 방법

노션에서 `앱 접근 비밀번호` 테이블을 열어 첫 행의 **`비밀번호`** 속성 값을 원하는 값으로 수정하면 즉시 반영됩니다. (앱/코드 수정 불필요)

## 배포 시 필요한 것 (1회)

1. **Worker 재배포**: `worker/` 폴더에서 `npx wrangler deploy`
   - 새 라우트 `POST /api/gate/verify` 와 자동 생성 로직이 포함됩니다.
2. 노션 **Integration 권한 확인**:
   - 테이블 자동 생성을 위해 통합에 콘텐츠 삽입(insert) 권한이 필요하고,
   - 식당 DB의 **부모 페이지가 통합과 공유**되어 있어야 합니다.
3. 첫 접근 시 자동 생성됩니다. 자동 생성이 불가한 환경이면 아래 "수동 준비"를 따르세요.

### (선택) 콜드스타트 지연 최소화

자동 생성 후 Worker 로그에 출력되는 DB ID를 `worker/wrangler.toml` 의 `PASSWORD_DB_ID` 에 등록하고 재배포하면, 매 요청마다 제목 검색을 하지 않아 조금 더 빠릅니다.

```toml
# worker/wrangler.toml
PASSWORD_DB_ID = "여기에-생성된-DB-ID"
```

### 수동 준비 (자동 생성이 막힌 경우)

1. 노션에서 데이터베이스(표)를 하나 만들고 제목을 `앱 접근 비밀번호` 로 지정
2. 텍스트 속성 `비밀번호` 를 추가하고 첫 행에 `53075` 입력
3. 이 DB를 통합과 공유
4. (선택) DB ID를 `PASSWORD_DB_ID` 에 등록 후 재배포

## 개발(dev) 참고

fail-closed 정책상 `VITE_API_BASE` 가 없는 로컬 dev 에서는 게이트를 통과할 수 없습니다.
로컬에서 게이트를 테스트하려면 `.env.local` 에 배포된 Worker 주소(`VITE_API_BASE`)를 넣고 실행하세요.
